Markdownでテキストを整える

Markdownは、Moodleで利用できるシンプルなテキストフォーマットを、強調 ( 太字、斜体 )、構造 ( 弾丸リスト、見出し ) およびリンク ( 画像または他のウェブリソース ) を簡単に追加することによって発展させます。あなたが選択したテキスト入力エリアの下にある書式ドロップダウンリストを単純に選択するだけで、Moodleの多くの場所でMarkdownを使用することができます。

基本概念

テキストを入力するには、単にテキストエリアまたはテキストボックスにテキストを打ち込んでください。段落の終わりと、次の行の始まりに空白行を残すため、リターンキーを2度押してください。

テキストの強調

Markdownでは、斜体のテキスト、太字のテキスト、または太字かつ斜体のテキストの、 3つのレベルのテキスト強調を追加することができます。テキストをアスタリスクで囲むことで、テキストを強調することができます。例:

*斜体* -> 斜体
**太字** -> 太字
***太字 斜体*** -> 太字 斜体

強調は、単一の単語、連続した単語、また単語の一部にも追加することができます。

*単一の* 単語 -> 単一の 単語
***連続した 単語*** -> 連続した 単語
**区別**できる -> 区別できる

この目的のために、アンダースコア ( _ ) を代わりに使用することができます。

見出し

Markdownでは、見出しを使用してテキストを分割することができます; 6つの異なるレベルの見出しを使用できますが、通常のテキストに3つ以上の見出しを使用することは不自然です。例えば、あなたが現在読んでいるテキストには、3つのレベルの見出しが使用されています。

行を1つまたはそれ以上のシャープ文字 ( # ) で開始することで、見出しを作成することができます。1つのシャープでは、一番大切な見出しを、6つのシャープでは最も重要でない、または一番小さな見出しを表示します。

# セクション見出し
## サブ・セクション見出し
### サブ・サブ・セクション見出し
その他

最初の2つのレベルの見出しは最も一般的で、テキストバージョンでさらに目立たせる代替的な方法として作成 ( 出力は前の方式と同じ ) することができます。次のようにタイトルの下に等号 ( = ) またはハイフン ( - ) の行を代わりに使用することができます:

セクション見出し
===============

サブセクション見出し
------------------

リスト

弾丸リスト

弾丸リストは、それぞれの行のコンテンツの前を、アスタリスクと空白で始めることで作成することができます。空白は重要ですので、入力を忘れないように注意してください。

* 1番目ポイント
* 2番目のポイント
* 3番目のポイント

は下記のようになります:

  • 1番目ポイント
  • 2番目のポイント
  • 3番目のポイント

番号付きリスト

同様に、番号付きリストは、それぞれの行を数字とそれに続く空白で始めて、関連するテキストを入力することで作成することができます。

1. 1番目ポイント
2. 2番目ポイント
3. 3番目ポイント

は下記のようになります:

  1. 1番目ポイント
  2. 2番目のポイント
  3. 3番目のポイント

インデントリスト

1つの構造の中に弾丸リストと番号付きリストがあっても、弾丸または番号付きリストをネスト、またはインデントすることができます: 

    * トップレベル弾丸1
      * サブ-弾丸  
      * サブ-弾丸2 
    * トップレベル弾丸2  
    1. 番号付きポイント1
       1. ネスト化した番号付きポイント  
    2. ネスト化した番号付きポイント2

は下記のようになります:

  • トップレベル弾丸1
    • サブ-弾丸
    • サブ-弾丸2
  • トップレベル弾丸2
    1. 番号付きポイント1
      1. ネスト化した番号付きポイント
    2. ネスト化した番号付きポイント2

引用段落

それぞれの行を鉤括弧 ( > ) で始めることで、引用欄を表示することができます。この文字は、多くのメールプログラムで引用欄を表示するために使用されています。一般的に出力は、引用欄を両方のマージンからインデントします。

> これは2行に展開した
> 引用段落です。

は下記のようになります:

これは2行に展開した引用段落です。

段落の最初の行に、1つの鉤括弧を付けるだけで保存することができます。

> これはすべて1つの
引用段落ですが、複数の行に
展開されています。

は下記のようになります:

これはすべて1つの引用段落ですが、複数の行に展開されています。

ウェブリンク

ウェブリソースのリンクを作成するには2つの方法があります: 1つ目は行の中に、次のインラインリンクを入れる方法です。まず、角括弧の中に読者にクリックして欲しいテキストを置きます。その直後に、表示したいページのURLを、間にスペースまたはギャップを入れずに丸括弧の中に置きます。リンクに関する任意のタイトルを、引用符を囲んでURLの後に追加することもできます。

文章の中の[サンプルリンク](http://example.com/ "任意のタイトル") です。

は下記のようになります:

文章の中の サンプルリンク です。

タイトルが設定された場合、ユーザがリンクテキスト上にマウスポインタを乗せると「ツールチップ」が表示されます。上記のリンクでお試しください。

長いリンクの場合、脚注スタイルを使用し、短い識別名 ( 短い説明語、語句または単純な番号 ) をリンク内の2番目の角括弧に入れることで、テキストの流れがバラバラになるのを避けることができます。

文章の中の [サンプルリンク][ex] です。

そして、実際にはリンクを設置した段落の直下、または文書の下部にある他のリンクURLが集められている部分が好ましいのですが、文章のどこにでも、idと関連するURLを定義することができます:

[ex]: http://example.com/ "任意のタイトル"

この2番目の例は、最初のものとは区別が付きません。これは単に作業中の文書を、更なる編集のために、きちんと整理するだけ ( 特にあなたが他の人と作業を行っている場合 ) の違いです。

最後のショートカットです。あなたのリンクテキストをURLと同じようにしたい場合は、次のようにURLを角括弧に入れてください:

<http://moodle.org/>

は下記のようになります:

http://moodle.org

画像

画像は、ウェブリンクと非常に似た方法で組み込まれますが、感嘆符により囲まれます。「altテキスト」 (altは代替物を意味します ) は、ユーザが様々な理由で画像を閲覧できない場合に提供されます。 従って、テキストは視覚的な合図が無い場合に、意味をなします。この処理をすることで、リマインダまたはヒントを、画像を目的としたMarkdownテキストの編集に提供します。

「タイトル」は、ユーザが画像の上にマウスポインタを置いた場合、小さなポップアップに表示され、付加的な情報を提供します。

![altテキスト](/path/img.jpg "任意のタイトル")

そして、ウェブリンクのように、すべてのURLを「脚注」スタイル参照と共に保存することで、あなたの文書を整理することができます。単に、画像に短い独自の名称を付けてください:

![altテキスト][photo]

そして、あなたの文書のどこでも良いですから、その名称と画像ファイルを関連付けてください:

[photo]: /url/to/img.jpg "任意のタイトル"

下記はサンプルおよびその出力です:

![Googleロゴ](http://www.google.com/images/logo.gif "Googleロゴ")

アクセスのしやすさのため重要な「altテキスト」は、しばしば画像へのリンクが壊れたり、一時的に利用不可の場合に使用されます:

![壊れた画像へのaltテキスト](http://example.com/intentionally.broken.link "この画像は決して表示されません。")
