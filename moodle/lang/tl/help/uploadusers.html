Pag-aaplowd ng mga user

Una, tandaan na hindi naman talaga kailangang mag-angkat ng mga user nang maramihan - upang mabawasan ang gawain mo sa pagmementina, dapat ay pag-aralan mo muna ang ibang anyo ng pagsusuri ng pagkakakilanlan na hindi nangangailangan ng mano-manong pagsasaayos. Tulad ng pagkonekta sa yari nang panlabas na database o pagpapahintulot sa mga user na lumikha ng sarili nilang account. Tingnan ang seksiyon hinggil sa Awtentikasyon sa mga pang-admin na menu.

Kung talagang nais mong mag-angkat ng maramihang user account mula sa isang text file, kakailanganin mong ipormat ang text file mo ng paganito:

  • Ang bawat linya sa file ay maglalaman ng isang record
  • Ang bawat record ay isang serye ng datos na pinaghihiwalay ng kuwit
  • Natatangi ang unang record ng file, at naglalaman ito ng listahan ng mga fieldname. Ito ang nagtatakda sa format ng kabuuang file.

    Mga kinakailangang fieldname: ang mga field na ito ay kailangang isama sa unang record, at bigyang kahulugan ayon sa bawat user

    username, password, firstname, lastname, email

    Mga default na fieldname: opsiyonal ang mga ito - kapag hindi ito isinama, ang mga halaga ay kukunin sa pangunahing admin

    institution, department, city, country, lang, auth, timezone

    Mga opsiyonal na fieldname: ganap na opsiyonal ang mga ito. Ang mga pangalan ng kurso ay ang "maiikling panglan" ng mga kurso - kung mayroon nito, ang user ay ieenrol bilang mag-aaral sa mga kursong itinakda. Ang mga pangalan ng pangkat ay dapat kaugnay ng angkop na kurso, a.b. group1 sa course1, atbp.

    idnumber, icq, phone1, phone2, address, url, description, mailformat, maildisplay, htmleditor, autosubscribe, course1, course2, course3, course4, course5, group1, group2, group3, group4, group5

  • Ang mga kuwit sa loob ng datos ay kailangang iencode na &#44 - awtomatikong idedecode ng script ang mga ito pabalik sa kuwit.
  • Para sa mga Boolean na field, gumamit ng 0 para sa ditotoo at 1 para sa totoo.
  • Tala: Kung ang user ay nakarehistro na sa Moodle user database, ihuhudyat ng script ang userid na bilang (database index) ng user na iyon, at ieenrol ang user bilang mag-aaral sa anumang itinakdang kurso NANG HINDI binabago ang iba pang itinakdang datos.

Halimbawa ng isang balidong file na pang-importa:

username, password, firstname, lastname, email, lang, idnumber, maildisplay, course1, group1
jonest, verysecret, Tom, Jones, jonest@someplace.edu, en, 3663737, 1, Intro101, Section 1
reznort, somesecret, Trent, Reznor, reznort@someplace.edu, en_us, 6736733, 0, Advanced202, Section 3
