URLs relatives, lecteurs multimédias et Hot Potatoes

Définitions

URL
L'adresse d'une ressource internet, telle qu'une page web, une image ou un fichier son.
URL absolue
Une URL qui commence par « http:// » est appelée URL absolue. Où qu'elle soit utilisée, elle pointe toujours vers la même ressource.
URL relative
Une URL qui ne commence pas par « http:// » est appelée URL relative. Elle peut être combinée avec l'URL absolue d'une page de « base » pour obtenir l'URL absolue de la ressource à désigner.

URLs relatives dans les tests Hot Potatoes

Les URLs relatives constituent une manière pratique de spécifier des images, des sons et des vidéos dans un test Hot Potatoes, parce qu'elle permettent au test d'être présenté et testé sur un ordinateur local avant d'être téléchargé sur un site Moodle. Lorsque le test est téléchargé sur le site Moodle, les fichiers multimédias doivent également être y téléchargés, afin qu'ils soient disponibles lorsque le test est administré et effectué depuis Moodle.

Pour des raisons de sécurité, Moodle ne permet l'accès direct aux fichiers d'un cours. Toutes les requêtes de fichiers d'un cours passent par script « gardien » qui vérifie que la personne demandant le fichier est inscrite au cours. Cependant, ce mécanisme peut casser les liens relatifs utilisés dans le test, parce que les règles habituelles pour obtenir une URL absolue à partir d'une URL relative ne fonctionnent pas.

La solution est simple : toutes les URLs relatives doivent être converties en URLs absolues au moment où le test est visualisé dans un navigateur. Dans le cas de tests Hot Potatoes, les URLs seront converties par Moodle, afin que les créateurs de contenus n'aient pas à se soucier de cette tâche pénible et source d'erreurs.

Lecteurs multimédias dans les tests Hot Potatoes

Certains lecteurs multimédias n'acceptent pas un fichier multimédia s'il provient du script qui gardien du cours. Dans cette situation, il est nécessaire de remplacer le lecteur multimédia par un autre qui est compatible with Moodle. Vous pouvez effectuer cette tâche vous-même en modifiant la référence au lecteurs multimédia dans le test, ou vous pouvez forcer Moodle à le faire en choisissant « Oui » pour l'option « Lecteurs multimédias compatibles » sur la page de configuration du test.
